ขั้นแรก สร้างโฟลเดอร์งาน และ nmp install prompt-sync ก่อนค่ะ
แล้วไปสร้าง repo เอา folder งานขึ้น git ไว้ก่อน

ลองเทสการเรียกใช้ prompt-sync ก่อน เพื่อให้แน่ใจว่าสามารถรับคำสั่งจากผู้ใช้ได้

===============================================================

ก่อนจะคิดโค้ด เราจะคิดแบบคนเล่นเกมก่อน:

เกมนี้คือให้เราเดินหาหมวก ระหว่างทางมีหลุม ห้ามตก แล้วก็ห้ามเดินออกนอกแผนที่
แอบคิดถึงพวกเกม pacman หรือเกม ระเบิด สมัยก่อนอยู่เมือนกันค่ะ น่าจะคล้ายๆกัน ว่าแล้วก็อยากเล่น :P

แปลว่าเกมนี้มี:

1. ตัวเรา
2. จุดหมาย (หมวก)
3. อันตราย (หลุม)
4. พื้นที่เดิน (map)

หลังจากเข้าใจว่าเกมมีองค์ประกอบอะไรบ้าง ก็คิดต่อว่าอะไรที่ต้องเก็บข้อมูลไว้ระหว่างเกม
เช่น ตำแหน่งผู้เล่นที่ต้องเปลี่ยนตลอดเวลา และแผนที่ที่ใช้ตรวจสอบว่าผู้เล่นเจออะไร เพราะเราจะต้องใช้ตัวแปรเพื่อเก็บข้อมูลเหล่านี้

แต่ก่อนจะสร้างตัวละคร ต้องสร้างแผนที่ก่อน ยังไม่ต้องเอาตัวละครไปวาง
เริ่มจากสร้างแผนที่เป็น array 2 มิติ และทดลองแสดงผลด้วย console.log
พบว่าข้อมูลแสดงออกมาเป็นรูปแบบ array ปกติ ทำให้ยังไม่เหมือนหน้าตาเกม
ขั้นตอนถัดไป ต้องหาวิธีแสดงผลให้เป็นตาราง

field เป็น array ที่เก็บ array ย่อยหลายตัว โดยแต่ละ array ย่อยแทน 1 แถวของแผนที่
จึงใช้ for loop เพื่อวนผ่านแต่ละแถว และแสดงผลทีละแถว โดยใช้ index (i) เพื่อเข้าถึงแต่ละตำแหน่งใน array

หลังจากสามารถแสดงแต่ละแถวได้แล้ว แต่ว่าละแถวยังเป็น array อยู่ ทำให้แสดงผลออกมาไม่สวย
คิดว่าควรแปลง array ให้เป็น string โดยใช้ .join() เพื่อรวมข้อมูลใน array และใส่ช่องว่างระหว่างแต่ละตัว ทำให้แผนที่แสดงผลออกมาในรูปแบบที่อ่านง่ายขึ้น
// เปลี่ยนเป็นใช้ emoji แทนเพื่อความน่ารัก สวยงาม และอ่านง่ายด้วยค่ะ 🌱🌱🌱🌱🌱

🤓 จากนั้นต้องมีตัวผู้เล่นในเกม จึงต้องสร้างตัวแปร playerRow และ playerCol เพื่อเก็บตำแหน่งของผู้เล่น เพราะแผนที่เป็นตารางเลยต้องมีทั้ง Row&Column

โดยกำหนดค่าเริ่มต้นให้อยู่ที่มุมซ้ายบนของแผนที่
และ update ค่าใน array เพื่อแสดงตัวผู้เล่นในตำแหน่งนั้น

ตอนนี้มี map&player แล้ว แต่ยังขยับไม่ได้ ต้องคิดว่าจะทำยังงัยให้ player ขยับได้ .........

ถ้าจะให้ตัวผู้เล่นขยับได้ จะต้องกด keyboard
ในที่นี่จะเลือกใช้ปุ่ม IJKL แทน เพราะมันอยู่ติดกัน และการจัดวางเหมือนกับปุ่มลูกศรบน keyboard จะได้กดง่ายขึ้นสำหรับคนเล่นจริงๆ

พอถึงขั้นตอนนี้ เราต้องเรียก ใช้ prompt ในการรับ input
และสร้างตัวแปร move ขึ้นมาเพื่อเก็บคำสั่งจากผู้เล่น

ก็เลยแก้ด้วยการเติม .toUpperCase(); เพื่อให้แปลงค่าเป็นตัวพิมพ์ใหญ่

===============================================================

เพิ่มเงื่อนไขเพื่อตรวจสอบว่าตำแหน่งใหม่อยู่ภายในขอบเขตหรือไม่ หากอยู่นอกขอบเขตจะจบเกมทันทีโดยใช้ break
ก่อนจะใช้ตำแหน่งใหม่ ต้องเช็คก่อนว่ามัน valid มั้ย มีเงื่อนไขที่ต้องเชคคือ
row < 0
row >= จำนวนแถว
col < 0
col >= จำนวนคอลัมน์

===============================================================

ตอนนี้จะเพิ่ม hole กับ hat ให้ครบก่อน คิดว่าจะทำแบบง่ายๆใส่ลงไปใน map เลยก่อนค่ะ
อยากให้ทำเสร็จ แล้วค่อยมาไล่เชค/แก้ไข/ปรับรูปแบบทีหลัง

\*\* ความจริงก็ทำแบบง่ายๆมาตั้งแต่เริ่มเลย ยังไม่ได้สร้าง class เลยค่ะ T_T
เดี๋ยวมาค่อยๆไล่ทำนะ หวังว่าจะทันเวลา สู้ๆนะตัวฉัน 🥹

หลังจากเพิ่มเงื่อนไขตรวจสอบตำแหน่งในแผนที่ไปแล้ว ตอนนี้จะเพิ่มการตรวจสอบตำแหน่ง hole กับ hat บ้าง
หากผู้เล่นตกหลุมจะจบเกม และหากพบหมวกจะชนะเกม
ใช้ตัวแปร currentPosition ให้อ่านและเข้าใจง่าย

ตอนนี้ทำเงื่อนไขหลักๆครบแล้ว (แหละมั้ง?) คิดว่าจะแปลงทั้งหมดให้เป็น class ก่อนจะดีกว่าค่ะ
กลัวตัวเองจะงงไปมากกว่านี้ ฮาาาา ...

ก่อนอื่น ต้องสร้าง class ขึ้นมาเพื่อรวมทุกอย่างของเกมไว้ใน object เดียว
เพิ่ม constructor เพื่อเก็บข้อมูลของแผนที่และตำแหน่งผู้เล่นไว้ภายใน class
แต่ยังไม่ลบโค้ดเดิม จะได้ทดสอบทีละขั้น

/
/

ตอนนี้สร้าง method print() ภายใน class เพื่อให้ class จัดการการแสดงผล
และเปลี่ยนจากการใช้ตัวแปรภายนอก มาใช้ข้อมูลภายใน class ผ่าน this.field

===============================================================

ต่อไปเอา player เข้าไปใน class ด้วย และเรียกใช้ก่อนจะ print ออกมา

แล้วก็ สร้าง movement ให้เป็น method

จากนั้นก็ย้าย loop และการรับ input เข้าไปใน class ด้วย
เพื่อให้ class เป็นผู้ควบคุมการทำงานของเกมทั้งหมด
และเรียกใช้ method ต่างๆภายใน class เพื่อจัดการ logic

เพิ่มเงื่อนไขตรวจสอบตำแหน่งในแผนที่ hole และ hat เข้าไปด้วย ให้ครบแบบที่ทำไว้ก่อนหน้านี้

พอเพิ่มเงื่อนไขทั้งหมดเข้า class แล้วก็เลยลบ code เก่าออก
